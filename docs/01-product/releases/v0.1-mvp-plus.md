## Zakres Funkcjonalny (Scope)

### 1. Audio Pipeline i Engagement
- [x] Architektura i decyzje technologiczne (ADR-0010)
- [ ] WebSocket audio ingest (binary chunks z przeglądarki)
- [ ] Audio feature extraction (librosa: RMS, spectral centroid, ZCR)
- [ ] Klasyfikacja zdarzeń (YAMNet via TFLite/ONNX)
- [ ] [Engagement score](../features/engagement-scoring.md) v1 (ważona suma z kalibracją)
- [ ] [Kalibracja per venue](../features/venue-calibration.md) (presety + ręczne korekty)

### 2. Setlista i Kontrola Czasu
- [ ] Model danych: setlisty, segmenty, warianty (full/short)
- [ ] [Import setlisty](../features/dynamic-setlist.md) (CSV)
- [ ] CRUD setlista (tworzenie, edycja, kolejność drag & drop)
- [ ] Show timeline (start/end/skip segment)
- [ ] [Kontrola czasu](../features/time-control.md): prognoza curfew, delta, scenariusze odzysku
- [ ] [Tagi operatora](../features/operator-tags.md) (quick tagi + custom)

### 3. Rekomendacje ML
- [ ] [LightGBM ranking](../features/ml-recommendations.md) (start: dane syntetyczne + reguły TINAP)
- [ ] Fallback regułowy (gdy ML confidence < threshold)
- [ ] Log rekomendacji i decyzji operatora
- [ ] Frontend: top 3-5 rekomendacji z accept/reject

### 4. Panel Operatora
- [ ] [Panel operatora](../features/operator-panel.md) — ciemny motyw, duże elementy dotykowe
- [ ] Tryb pre-show (setup: setlista, venue, kalibracja, audio test)
- [ ] Tryb live (engagement gauge, timeline, rekomendacje, czas, tagi)
- [ ] Tryb post-show (analytics, timeline, tabela segmentów)
- [ ] WebSocket real-time updates + reconnect strategy

### 5. Post-show i Raporty (Wariant B)
- [ ] [Panel post-show](../features/post-show-analytics.md) (engagement timeline, tabela, log decyzji)
- [ ] [Eksport danych](../features/data-export-reports.md) (CSV, JSON)
- [ ] [Automatyczne raporty](../features/data-export-reports.md) (PDF)

### 6. Infrastruktura
- [ ] Monorepo: `apps/api` (FastAPI), `apps/web` (React), `infra/` (Docker)
- [ ] Docker Compose (api + web + postgres + redis)
- [ ] GitHub Actions (lint + test na PR)
- [ ] VPS deployment (Hetzner/DigitalOcean)
- [ ] Prosty auth (API key lub JWT, jeden operator)

## Kryteria Sukcesu Pilota

- Stabilność 90+ minut.
- Engagement koreluje z obserwowaną energią (ocena TINAP).
- Prognoza curfew ±2 min.
- Rekomendacje "mają sens" wg showcallera.
- Panel czytelny w backstage.
- Reconnect < 10 sekund.
