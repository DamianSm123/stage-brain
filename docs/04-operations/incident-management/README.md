# Incident Management

Proces zarządzania incydentami w OpsDesk ma na celu szybkie przywrócenie działania usług oraz naukę na błędach.

## Fazy Incydentu
1.  **Wykrycie**: Alert z monitoringu lub zgłoszenie użytkownika.
2.  **Reakcja**: Osoba On-Call podejmuje działania, ocenia priorytet i komunikuje status.
3.  **Mitygacja**: Przywrócenie działania usługi (np. rollback, restart, obejście problemu).
4.  **Rozwiązanie**: Trwała naprawa przyczyny źródłowej (może nastąpić później).
5.  **Postmortem**: Analiza i wnioski.
